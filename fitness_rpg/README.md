# 🏋️ Fitness RPG - Standalone Desktop Application

Приложение для трекинга физических упражнений с RPG-системой прогрессии, детализированными зонами тела и реалистичной механикой потери прогресса.

## 📁 Структура проекта

```
fitness_rpg/
├── app/                          # Frontend (HTML/CSS/JS)
│   ├── index.html               # Главный HTML файл
│   ├── styles.css               # Стили в стиле "Old Manuscript"
│   └── main.js                  # Интерактивная логика UI
├── backend/                      # Python backend
│   ├── models.py                # Pydantic модели данных
│   ├── progression.py           # Математика прогрессии и декремента
│   ├── database.py              # SQLite обёртка
│   ├── exercises_db.json        # База из 48 упражнений
│   ├── config.json              # Конфигурация баланса
│   └── database_schema.sql      # Схема БД
├── docs/                         # Документация
│   └── MATHEMATICAL_MODELS.md   # Подробное описание формул
└── README.md                     # Этот файл
```

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
cd fitness_rpg/backend
pip install pydantic
```

### 2. Инициализация базы данных

```bash
python database.py
```

### 3. Запуск тестов прогрессии

```bash
python progression.py
```

### 4. Открытие UI

Просто откройте `app/index.html` в браузере или используйте локальный сервер:

```bash
cd app
python -m http.server 8080
# Откройте http://localhost:8080
```

## 📐 Математические модели

### Порог XP для уровня L
```
XP_threshold(L) = BASE_XP × (L ^ 1.6)
```
Где BASE_XP = 100 (настраивается в config.json)

### Общий уровень (с пенальти за дисбаланс)
```
Harmonic_Mean = N / Σ(1 / (level_i + 0.5))
Variance_Penalty = 1 - 0.25 × (σ_levels / max_level)
Overall_Level = floor(Harmonic_Mean × Variance_Penalty)
```

### Декремент (потери при простое)
- **Дни 0-1**: 0% потерь (восстановление)
- **День 2**: 5% от current_XP (мягкий старт)
- **Дни 3+**: `daily_loss = current_XP × (0.04 + 0.015 × log₂(level + 1))`

**Защитный буфер**: XP никогда не падает ниже `max(10% от порога уровня, 50 XP)`

## ⚙️ Конфигурация баланса

Все параметры вынесены в `backend/config.json`:

```json
{
  "progression": {
    "base_xp": 100,
    "exponent": 1.6
  },
  "decay": {
    "grace_period_days": 1,
    "soft_decay_rate": 0.05,
    "base_decay_rate": 0.04,
    "level_scaling_factor": 0.015,
    "min_buffer_percent": 0.10
  }
}
```

## 🎮 Использование

1. **Клик по зоне тела** → отображение деталей (уровень, XP, прогресс)

3. **Соединительные линии** → анимированные линии от панели к выбранной зоне

## 🔧 Настройка баланса

### Для более быстрой прогрессии:
- Уменьшите `exponent` до 1.4-1.5
- Увеличьте `base_xp` до 150-200

### Для более медленного декремента:
- Уменьшите `base_decay_rate` до 0.02-0.03
- Увеличьте `grace_period_days` до 2-3

### Для масштабирования до 200+ уровня:
- Уменьшите `exponent` до 1.3-1.4
- Увеличьте `base_xp` пропорционально

## 📝 Рекомендации по балансу

1. **Калибровка BASE_XP**: Начните с 100, отслеживайте среднее время достижения уровня 10. Целевое значение: 4-6 недель при регулярных тренировках.

2. **Настройка decay_rate**: Период полураспада должен составлять 14-18 дней на высоких уровнях. Если игроки теряют мотивацию — уменьшите rate.

3. **Пенальти дисбаланса**: Коэффициент 0.25 даёт умеренный штраф. Для более жёсткого баланса увеличьте до 0.3-0.35.

4. **Периодизация**: Рассмотрите добавление weekly/monthly бонусов за консистентность тренировок.

## 🛠 Расширение функционала

### Добавление нового упражнения:
1. Откройте `backend/exercises_db.json`
2. Добавьте запись в массив `exercises`:
```json
{
  "name": "New Exercise",
  "type": "strength",
  "base_xp_per_rep": 2.0,
  "zones": {"legs": 0.8, "core": 0.3},
  "beginner_params": {"sets": 3, "reps": 12, "rest_seconds": 60},
  "notes": "Description here"
}
```

### Добавление новой зоны:
1. Обновите SQL схему в `database_schema.sql`
2. Добавьте маппинг в `main.js` (ZONE_MAPPING)
3. Добавьте SVG элемент в `index.html`
