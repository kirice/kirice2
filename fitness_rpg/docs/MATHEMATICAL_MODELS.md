# 📐 МАТЕМАТИЧЕСКИЕ МОДЕЛИ И ФИЗИОЛОГИЧЕСКОЕ ОБОСНОВАНИЕ

## 1. ЗОНЫ ТЕЛА И УРОВНИ

### Формула порога XP для уровня L:
```
XP_threshold(L) = BASE_XP * (L ^ EXPONENT)
```
Где:
- `BASE_XP = 100` (калибруется в config.json)
- `EXPONENT = 1.6` (подобран для плавной прогрессии)

**Пример расчёта:**
| Уровень | Порог XP | Накопительный XP |
|---------|----------|------------------|
| 1       | 100      | 100              |
| 2       | 303      | 403              |
| 3       | 580      | 983              |
| 4       | 930      | 1913             |
| 5       | 1360     | 3273             |
| 6       | 1878     | 5151             |
| 7       | 2480     | 7631             |
| 8       | 3172     | 10803            |
| 9       | 3960     | 14763            |
| 10      | 4850     | 19613            |

**Физиологическое обоснование:** Экспоненциальный рост отражает закон убывающей отдачи - чем выше уровень, тем больше усилий требуется для прогресса.

## 2. РАСЧЁТ XP ЗА УПРАЖНЕНИЕ

### Формула:
```
XP = (sets × reps × intensity_factor) × base_xp_per_rep × zone_coefficient
```

Где:
- `intensity_factor`: 
  - Новичок: 1.0
  - Средний: 1.3
  - Продвинутый: 1.6
- `base_xp_per_rep`: базовое значение упражнения (из exercises_db.json)
- `zone_coefficient`: коэффициент влияния на конкретную зону тела

**Пример:** Приседания (3 подхода × 12 повторений, новичок)
```
XP_Legs = (3 × 12 × 1.0) × 2.5 × 1.0 = 90 XP
XP_Core = (3 × 12 × 1.0) × 2.5 × 0.4 = 36 XP
XP_Back = (3 × 12 × 1.0) × 2.5 × 0.2 = 18 XP
```

## 3. ОБЩИЙ УРОВЕНЬ (PENALTY FOR IMBALANCE)

### Modified Harmonic Mean с множителем дисперсии:

```
Step 1: Harmonic_Mean = N / Σ(1 / (level_i + 0.5))

Step 2: Variance_Penalty = 1 - 0.25 × (σ_levels / max_level)

Step 3: Overall_Level = floor(Harmonic_Mean × Variance_Penalty)
```

Где:
- `N` = количество зон (10)
- `level_i` = уровень i-й зоны
- `σ_levels` = стандартное отклонение уровней
- `max_level` = максимальный уровень среди всех зон
- `+0.5` = сглаживающий член для устойчивости к нулям

**Пример расчёта:**
Зоны: Legs=10, Back=1, остальные=5 (N=8)
```
Harmonic_Mean = 8 / (1/10.5 + 1/1.5 + 6/5.5) ≈ 4.8
σ = √[((10-5.5)² + (1-5.5)² + 6×(5-5.5)²) / 8] ≈ 2.8
Variance_Penalty = 1 - 0.25 × (2.8 / 10) ≈ 0.93
Overall_Level = floor(4.8 × 0.93) ≈ 4
```

**Физиологическое обоснование:** Гармоническое среднее естественным образом "тянет вниз" общий уровень при наличии слабых зон, что соответствует принципу лимитирующего фактора в физиологии.

## 4. СИСТЕМА ДЕКРЕМЕНТА (ДЕТРЕНИНГ)

### Логика потерь по дням простоя:

| Дней простоя | Потери | Обоснование |
|--------------|--------|-------------|
| ≤ 1          | 0%     | Краткосрочное восстановление |
| 2            | 5%     | Начало детренинга |
| ≥ 3          | Progressive decay | Активная потеря адаптации |

### Формула ежедневной потери (для days ≥ 3):
```
daily_loss = current_XP × (0.04 + 0.015 × log₂(current_level + 1))
```

**Защитный буфер:**
```
min_remaining_XP = max(10% × XP_threshold(current_level), 50 XP)
```

**Пример:** Уровень 7, текущий XP = 2000
```
daily_loss = 2000 × (0.04 + 0.015 × log₂(8))
           = 2000 × (0.04 + 0.015 × 3)
           = 2000 × 0.085 = 170 XP/день

min_buffer = max(0.10 × 2480, 50) = 248 XP
```

**Период полураспада:** ~14-18 дней на высоких уровнях, быстрее на низких.

**Физиологическое обоснование:** Принцип обратимости - адаптации теряются при отсутствии стимула. Логарифмическая зависимость отражает, что более тренированные атлеты теряют прогресс медленнее (большая "мышечная память").

## 5. КОНФИГУРАЦИЯ БАЛАНСА (config.json)

Все коэффициенты вынесены в конфигурационный файл для лёгкой калибровки:

```json
{
  "progression": {
    "base_xp": 100,
    "exponent": 1.6,
    "smoothing_constant": 0.5
  },
  "decay": {
    "grace_period_days": 1,
    "soft_decay_day": 2,
    "soft_decay_rate": 0.05,
    "base_decay_rate": 0.04,
    "level_scaling_factor": 0.015,
    "min_buffer_percent": 0.10,
    "min_buffer_absolute": 50
  },
  "overall_level": {
    "variance_penalty_coefficient": 0.25
  }
}
```
