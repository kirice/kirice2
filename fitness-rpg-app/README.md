# Fitness RPG Desktop Application

Настоящее desktop-приложение для фитнес-трекинга с RPG-системой прогрессии.

## 🎯 Особенности

- **Полностью автономное приложение** - один .exe файл, не требует браузера
- **RPG-система уровней** - 10 зон тела с индивидуальной прогрессией
- **Математическая модель** - гармоническое среднее с пенальти за дисбаланс
- **Система декремента** - реалистичная потеря прогресса при простое
- **48 упражнений** - полная база с коэффициентами для каждой зоны
- **Винтажный стиль** - интерфейс в стиле old manuscript

## 🚀 Установка и запуск

### Требования

1. **Node.js** 18+ - https://nodejs.org/
2. **Rust** - https://rustup.rs/
3. **Visual Studio Build Tools** (Windows) или Xcode (macOS)

### Шаг 1: Установка зависимостей

```bash
cd fitness-rpg-app
npm install
```

### Шаг 2: Запуск в режиме разработки

```bash
npm run tauri dev
```

Приложение откроется в отдельном окне (не в браузере!).

### Шаг 3: Сборка релизной версии (.exe)

```bash
npm run tauri build
```

После сборки .exe файл будет находиться в:
- Windows: `src-tauri/target/release/fitness-rpg-app.exe`
- macOS: `src-tauri/target/release/bundle/macos/`
- Linux: `src-tauri/target/release/bundle/deb/` или `appimage/`

## 📐 Математические модели

### 1. Порог XP для уровня L

```
XP_threshold(L) = BASE_XP × (L ^ 1.6)
```

Где BASE_XP = 100

| Уровень | Требуется XP |
|---------|-------------|
| 1       | 100         |
| 2       | 303         |
| 3       | 580         |
| 4       | 930         |
| 5       | 1360        |
| 10      | 4850        |
| 50      | 52,480      |
| 100     | 158,489     |

### 2. XP за упражнение

```
XP = (sets × reps × intensity_factor) × base_xp_per_rep × zone_coefficient
```

- **intensity_factor**: 1.0 (новичок), 1.3 (средний), 1.6 (продвинутый)
- **base_xp_per_rep**: базовое значение упражнения (0.4 - 3.0)
- **zone_coefficient**: коэффициент зоны (0.2 - 1.0)

**Пример**: Приседания 3×10, интенсивность 1.0
- Legs: 3 × 10 × 1.0 × 2.5 × 1.0 = 75 XP
- Core: 3 × 10 × 1.0 × 2.5 × 0.4 = 30 XP
- Back: 3 × 10 × 1.0 × 2.5 × 0.2 = 15 XP

### 3. Общий уровень (Modified Harmonic Mean)

```
Overall_Level = floor( N / Σ(1 / (level_i + 0.5)) × (1 - 0.25 × (σ_levels / max_level)) )
```

Где:
- N = количество зон (10)
- σ_levels = стандартное отклонение уровней
- max_level = максимальный уровень среди зон

**Почему гармоническое среднее?**
- Естественным образом "тянет вниз" при наличии слабых зон
- Добавочный член +0.5 предотвращает деление на ноль
- Множитель дисперсии дополнительно штрафует за перекос

**Пример расчёта**:
- Зоны: Legs=10, Back=1, остальные=5 (N=8)
- Гармоническое среднее ≈ 4.8
- Пенальти за дисперсию ≈ 0.18
- Итоговый уровень = floor(4.8 × 0.82) = 3

### 4. Декремент (детренинг)

```
daily_loss = current_XP × (0.04 + 0.015 × log₂(current_level + 1))
```

**Правила**:
- Days ≤ 2: 0% потерь (период толерантности)
- Days ≥ 3: прогрессивные потери
- Минимальный остаток: max(10% от порога уровня, 50 XP)

**Пример**:
- Уровень 10, XP = 4000
- daily_loss_rate = 0.04 + 0.015 × log₂(11) ≈ 0.093
- Через 5 дней простоя: 4000 × (1 - 0.093)^3 ≈ 2960 XP

## 🎮 Использование

### Запись тренировки

1. Выберите упражнение из списка
2. Укажите количество подходов и повторений
3. Выберите уровень интенсивности
4. Нажмите "ЗАПИСАТЬ"

XP будет автоматически распределён по соответствующим зонам.

### Просмотр прогресса

- **Клик** на зону тела → детальная информация
- **Наведение** на зону → всплывающая подсказка с уровнем и XP
- **Правая панель** → список всех зон с уровнями

### Тестирование декремента

Нажмите кнопку "Применить декремент (тест)" для симуляции потерь XP.

## ⚙️ Настройка баланса

Все параметры вынесены в код (`src-tauri/src/lib.rs`) в структуру `Config`:

```rust
pub struct Config {
    pub base_xp: f64,              // 100.0 - базовый XP
    pub exponent: f64,             // 1.6 - экспонента роста
    pub decay_start_days: i32,     // 2 - дни без потерь
    pub daily_decay_base: f64,     // 0.04 - базовая скорость потерь
    pub daily_decay_level_mult: f64, // 0.015 - множитель от уровня
    pub min_remaining_percent: f64,  // 0.1 - мин. % от порога
    pub min_remaining_absolute: f64, // 50.0 - мин. абсолютный XP
}
```

### Рекомендации по балансировке

1. **BASE_XP**: 
   - Увеличьте до 150-200 для более медленного прогресса
   - Уменьшите до 50-80 для быстрого ощущения роста

2. **exponent**:
   - 1.5-1.7: стандартный рост
   - >1.8: очень сложно достигать высоких уровней
   - <1.4: слишком быстрый прогресс на высоких уровнях

3. **decay_rate**:
   - Увеличьте `daily_decay_base` до 0.06-0.08 для жёсткого режима
   - Включите `vacation_mode` в коде для приостановки потерь

## 🔧 Технические детали

### Команды Tauri

| Команда | Описание |
|---------|----------|
| `get_zones` | Получить список всех зон |
| `get_zone_progress` | Прогресс конкретной зоны |
| `get_all_progress` | Прогресс всех зон |
| `log_exercise` | Записать тренировку |
| `calculate_overall` | Рассчитать общий уровень |
| `apply_decay_tick` | Применить декремент |
| `get_exercises` | Получить список упражнений |

### Хранение данных

Данные хранятся в SQLite базе `fitness_rpg.db` в той же директории, что и .exe файл.

**Таблицы**:
- `zones` - названия зон
- `zone_progress` - текущий XP, уровень, дата последней тренировки
- `workout_logs` - история тренировок
